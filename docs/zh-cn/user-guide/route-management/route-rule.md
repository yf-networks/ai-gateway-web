# 路由规则

## 概述

### 概念和原理

对于AI网关的流量，可能需要由不同的AI业务集群来服务。

对于某个请求，AI网关会根据配置的路由规则表达式依次匹配；命中后，将请求转发到该规则指定的目标集群。

每条路由规则包含一个**表达式**匹配条件和一个**目标集群**，即满足匹配条件的请求会被转发到该目标集群。规则按列表顺序依次匹配，命中后停止。

### 路由规则列表

- 在左侧菜单进入"路由管理"->"路由规则"页。

![路由规则列表页](../../../images/AI-gateway-rule-config.png)

路由规则列表页包含以下功能：

- **添加规则**：点击后进入规则创建页面。
- **提交并生效**：规则修改完成后，点击该按钮使配置生效。
- **搜索**：可按表达式、目标集群、描述进行筛选查询。
- **规则列表**：展示已有规则的表达式、目标集群、描述和操作（编辑、删除）。
- **规则排序**：支持拖动表格行调整规则生效顺序。

### 添加/编辑路由规则

- 点击"添加规则"，进入规则配置页面。

![路由规则配置页](../../../images/AI-gateway-rule-config-detail.png)

添加/编辑路由规则时，需要配置以下内容：

- **表达式**（必填）：通过表达式编辑器构建请求匹配条件。可点击逻辑连接符（`(`、`)`、`&&`、`||`、`!`）和匹配条件标签，在表达式编辑区生成并组合匹配规则，按需修改参数即可。
- **目标集群**（必填）：选择请求匹配后转发的目标AI业务集群。
- **描述**（可选）：填写该路由规则的说明信息。

配置完成后，点击**本地保存**保存当前规则，或点击**重置**清空当前填写内容。

### 提交并生效

在路由规则列表页完成所有规则的新增、编辑、删除或排序后，点击**提交并生效**，使配置正式生效。
